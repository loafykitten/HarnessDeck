# Claude Command — Plan

A local "control center" web app for managing Claude Code on this machine.
Starts at login, serves on `http://localhost:4553`, bound to **127.0.0.1 only**.

## Feature set

### 1. Dashboard
- **Greeting header** — personalized greeting: display name + local weather
  (zip code → Open-Meteo, free/no key) + a kind/cute/whimsical one-liner
  generated by **Haiku** (headless `claude -p --model haiku`, so it rides the
  existing subscription auth — no API key). Name/zip configurable in Config.
  Greeting regenerated on a slow cadence (e.g. hourly), cached.
- **Project grid** — every folder in `~/Developer` is a project. Card shows name,
  last activity (from `~/.claude/projects/<slug>` transcript mtimes), running
  session count.
- **Active sessions widget** — projects with running sessions, one click to jump
  into any session tab. This is the primary navigation for in-work projects.
- **Usage widgets**
  - 5-hour block: % used + reset time
  - Weekly: % used + reset time
  - Subscription month: total tokens burnt + **API-equivalent cost** in USD
  - **Plan card**: subscription tier + monthly renewal date

### 2. Project view
- **Tabbed terminal sessions** per project (xterm.js). Each tab is one Claude
  Code session; multiple tabs per project for parallel work.
- **New session** — spawns `claude` inside a fresh tmux session in the project
  directory; the web terminal attaches.
- **Image paste** — paste an image into the terminal pane → frontend intercepts
  the paste event, POSTs the blob to the backend, backend saves it to a temp
  file and writes the file path into the PTY (Claude Code reads image paths
  natively). Works for screenshots and copied images.
- Rename / close (kills the tmux session) per tab.

### 3. Skills view
- List skills from `~/.claude/skills` (name + description parsed from
  SKILL.md frontmatter), plus plugin-provided skills read-only.
- **Install by URL** — git repo, zip, or raw folder URL → cloned/unpacked into
  `~/.claude/skills/<name>`.
- **Generate via prompt** — describe the skill; backend runs a headless
  `claude -p` job that scaffolds SKILL.md (+ assets) into a new skill folder,
  shown for review before saving.
- **Drill-in editor** — edit SKILL.md and supporting files in-browser, saved
  directly to disk.

### 4. Config view
- `~/.claude/settings.json` — form controls for common fields (default model,
  permissions, statusline) plus a raw JSON editor with validation.
- `~/.claude/CLAUDE.md` — in-browser markdown editor. (Per-project CLAUDE.md
  editing from the project view later.)
- **Greeting preferences** — display name, zip code (weather), greeting on/off.
  Stored in the app's own config file, not Claude's.
- Changes are written immediately; new Claude sessions pick them up.

### 5. Themes & chrome
Two themes, both dark neon/vaporwave gradients, via CSS custom properties,
toggle persisted in localStorage:
- **pink/blue** (default)
- **pink/red**

Design language (converged from draft round 1): **glassmorphic panels over a
bright drifting gradient with glow orbs** (draft #1) + **neon glow borders and
luminous numerals** (draft #6). Navigation is a **left icon rail sidebar**,
expandable to show labels. Terminal tabs fuse visually with the terminal pane
(draft #2's treatment). No card-within-a-card nesting; every widget's purpose
is self-evident.

## Stack

| Layer | Choice | Why |
|-------|--------|-----|
| Runtime + backend | **Bun** (`Bun.serve`, native WebSockets) | Fast startup, TS with no build step, first-class WS |
| Frontend | **Svelte 5 + Vite SPA** | Light (~15KB runtime), reactive, app-like; persistent terminal tabs need an SPA |
| Terminal | **xterm.js** (+ fit & webgl addons) | Standard, fast |
| Sessions | **tmux** (`claudecmd-<project>-<n>` sessions) | Sessions survive server restarts, reboots of the web app, closed browsers. *Requires `brew install tmux`.* |
| PTY bridge | `bun-pty` (portable-pty via FFI) | node-pty is unreliable under Bun. Fallback if flaky: tiny Node sidecar for PTY only. |
| Usage data | `bunx ccusage --json` (blocks/weekly/monthly) | Parses `~/.claude/projects/**/*.jsonl`; gives tokens + API-equivalent cost per model. |
| Limits/resets | Anthropic OAuth usage endpoint, creds from macOS Keychain ("Claude Code-credentials") | Authoritative 5h/weekly utilization + reset times; fallback = local estimate from ccusage blocks. |
| Startup | launchd LaunchAgent `~/Library/LaunchAgents/com.fenn.claudecommand.plist` (`RunAtLoad` + `KeepAlive`) | Native macOS "run at startup" |

## API sketch

REST
- `GET  /api/projects` — folders in ~/Developer + activity + session counts
- `GET/POST/DELETE /api/sessions` — list / create (project, name) / kill
- `GET  /api/usage` — 5h block, weekly, month tokens, API-equivalent cost
- `GET/PUT /api/skills/:name` + `POST /api/skills/install` (url | prompt)
- `GET/PUT /api/config/settings` and `/api/config/claude-md`
- `POST /api/sessions/:id/image` — pasted image → temp file → path into PTY

WebSocket
- `/ws/session/:id` — terminal I/O (attach to tmux via PTY)
- `/ws/events` — push: session list changes, usage ticker

## Security
Loopback-only bind, no auth in v1 (single-user machine). This server executes
arbitrary code by design — it must never listen beyond 127.0.0.1.

## Implementation notes (learned the hard way)
- tmux mangles control chars (incl. `\t`) to `_` in `-F` output — use `|` separators.
- `set-option -t` rejects the `=exact` target prefix; use `name:`.
- Bun.serve default `idleTimeout` is 10s — greeting generation must be
  async-cached, never awaited in a request.
- bun-pty's resize propagates through `tmux attach` to the window size
  automatically (window-size follows latest client) — no manual resize-window.
- ccusage counts codex/opencode agents too — filter modelBreakdowns to
  `claude*` for the dashboard numbers.
- The credentials file's `subscriptionType` goes STALE after plan changes.
  `api.anthropic.com/api/oauth/profile` is authoritative: plan tier
  (`organization.rate_limit_tier`), renewal anniversary
  (`organization.subscription_created_at`), and account display name.
