# Claude Command — Design Brief, Round 2 (convergence)

You are producing ONE design draft for "Claude Command", a local web control
center for managing Claude Code. Round 1 produced 10 explorations (now in
`drafts/archive/`); the user gave detailed feedback. Round 2 converges on it.

## The verdict from round 1 — this defines the design language

**Base: archive/design-1.html (Glassmorphic Neon) — the clear favorite.**
READ IT before you write anything; you are evolving it, not starting over.
Loved about it: the frosted glass over a bright gradient with subtle glow
orbs; the modern colorful accents (gradient nav underline, colorful graphs,
the green "active" status text on projects); the clean project grid; the
colorful terminal syntax; the greeting.

**Add: the GLOW of archive/design-6.html.** The user: "I'm in love with the
glow… I love glow!" Neon gradient borders, luminous stat numerals, colored
glow shadows. Layer this onto the glass language generously — but readability
still wins.

**Terminal tabs: steal the treatment from archive/design-2.html** — the user
specifically loved how its terminal session tabs work (tabs that feel
attached/fused to the terminal pane). Recolor it to the glass language.

**Active-sessions card: archive/design-7.html and design-8.html got this
right** — a strong, scannable card with live session rows. Use that idea, BUT:

**Banned (explicit user dislikes):**
- Card-within-a-card nesting ("really dislike GPT's card-within-a-card").
  Rows + hairline dividers inside one glass card, never boxes in boxes.
- Glass-outline-on-glass for the active nav state (design-1's one flaw) —
  active nav needs a solid/gradient treatment that reads instantly.
- Unexplained widgets (design-1's "live [time]" widget confused). Every
  widget's purpose must be self-evident from its label/content.
- Jargon like "usage envelope". Plain language only.
- Business/corporate/salesy vibes; data-dense mission-control layouts.

## Structural changes from round 1

1. **Left sidebar navigation, icon-only rail** (inline SVG icons: Dashboard,
   Projects, Skills, Config + theme toggle at the bottom). ~64–72px wide,
   tooltips on hover, and an expand affordance (chevron/pin) that widens it
   to show labels — wire the expand/collapse in JS, it's part of what's
   being evaluated. Active item: gradient fill + glow (not a glass outline).
2. **Greeting header** on the dashboard (keep from design-1, make it warmer):
   greeting + name + weather + a whimsical AI-generated line. Mock content:
   "Good evening, Fenn" · "74° · clear skies in Portland" · italic sub-line:
   "the neon's humming, the tokens are cheap, and SwiftClaw missed you ✦"
   (In the real app this line is generated by Claude Haiku from name, weather,
   and activity; name/zip are configurable.) Add a tiny gear icon on the
   greeting hinting it's configurable — dead control.
3. **New plan widget** alongside usage: "Claude Max 20×" tier badge,
   "renews Aug 3". Plain, glanceable.
